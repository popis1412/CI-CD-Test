# 게임 QA 자동화 트레이닝 키트 (Windows + Jenkins 통합 버전 ) - v1
이 키트는 Jenkins를 활용하여 QA 테스트 프로세스를 자동화하고, 테스트 결과 리포트를 생성 및 관리하는 실습 환경을 제공합니다. Postman과 같은 외부 도구 없이도 윈도우 환경에서 독립적으로 작동하도록 설계되었습니다.

# 주요 구성 요소
- Jenkinsfile: 전체 자동화 파이프라인 스크립트 (리포트 생성, 로컬 저장, 슬랙 알림 제어)

- 일곱개의 대죄 _ 오리진 테스트 케이스.xlsx: 테스트 시나리오 및 예상 결과, 실제 결과가 기록된 테스트 케이스 명세서

- 일곱 개의 대죄_ 오리진 역기획서.pdf: 게임의 시스템과 로직을 분석한 역기획서, 테스트 케이스 작성 및 검증 로직의 근거 자료로 사용된다.

# 시작하기
1) 파이프라인 생성
 1. Jenkins 대시보드에서 '새로운 Item' 클릭 -> 'Pipeline' 선택.
 2. 설정 메뉴의 Pipeline 섹션에서:
	Definition: 'Pipeline script from SCM' 선택.
	SCM: 'Git' 선택 후 저장소 URL 입력.
	Script Path: Jenkinsfile 확인.

 2) 빌드 실행
  '지금 빌드'를 클릭하면 자동화 프로세스가 시작됩니다.
  소스 코드 체크아웃 -> 테스트 시뮬레이션 -> 리포트 생성 -> 결과 공지 순으로 진행됩니다.

# 테스트 결과 확인 방법
1) 로컬 저장소 (영구 보관)
빌드가 완료되면 젠킨스 서버가 설치된 PC의 아래 경로에 리포트가 자동으로 생성됩니다.

저장 경로: 
	저장경로\newman

생성 파일:
	QA_Automation_Report.html (시각화 리포트)
	result.txt (텍스트 요약본)

2) Jenkins 웹 화면
프로젝트 메뉴의 **'7DS Origin QA Automation Reports'**를 클릭하여 HTML 리포트를 즉시 확인할 수 있습니다.

3) 슬랙(Slack) 알림
빌드 성공/실패 시 설정된 슬랙 채널로 결과 메시지가 전송됩니다.
메시지에 포함된 링크를 통해 리포트 확인 페이지로 바로 이동할 수 있습니다.

# 자동화 파이프라인 단계
Prepare: 이전 빌드 데이터 정리 및 폴더 생성.
Run QA Tests: [전투] 합기 테스트 및 [모험] 기믹 테스트 시뮬레이션.
Generate Report: 윈도우 배치 스크립트를 이용해 로컬 절대 경로에 리포트 출력.
Post Actions: 리포트 아카이빙 및 슬랙 알림 전송.

# 주의 사항
본 환경은 윈도우 환경에 최적화되어 있습니다.
저장 경로에 대한 쓰기 권한이 필요합니다.
슬랙 알림을 받으려면 Jenkins 시스템 설정에서 슬랙 플러그인 설정이 완료되어야 합니다.
