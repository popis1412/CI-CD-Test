# 📑 Project: Python & Jenkins 기반 QA 테스트 리포팅 자동화 파이프라인

본 프로젝트는 엑셀(Excel)에 기록된 대규모 테스트 케이스 데이터를 **Python**으로 정밀 분석하고, **Jenkins** CI/CD 파이프라인을 통해 분석 결과와 시각화된 리포트를 **Slack**으로 자동 전파하는 실무형 QA 자동화 시스템입니다.

---

## 1. 🚀 포트폴리오 개요 (Why GitHub & Jenkins?)

단순한 스크립트 실행을 넘어 **GitHub**와 **Jenkins**를 활용하여 포트폴리오를 구성한 이유는 다음과 같습니다.

* **지속적 통합(CI)의 실무적 이해:** 테스트 결과(`QA_Test.xlsx`)가 업데이트되어 GitHub에 Push되는 순간을 자동화의 기점(Trigger)으로 설정하여, 현업의 '지속적 테스트' 흐름을 구현했습니다.
* **데이터의 중앙 관리:** 팀 단위 업무 환경에서 데이터의 변경 이력을 추적하고, 누구나 최신 품질 리포트에 즉시 접근할 수 있는 환경을 제공합니다.
* **프로세스 자동화 역량:** 반복적인 수동 집계 업무를 자동화 파이프라인(Pipeline as Code)으로 구축하여 공정 효율을 높이는 엔지니어링 능력을 증명합니다.

---

## 2. 🏗 파이프라인 구조 및 산출물의 의미

전체 파이프라인은 4가지 핵심 스테이지로 구성되며, 각 단계는 품질 관리 공정에서 명확한 목적과 가치 있는 산출물을 생성합니다.



| 파이프라인 단계 | 수행 내용 (Test Logic) | 주요 산출물 | 산출물의 의미 및 가치 |
| :--- | :--- | :--- | :--- |
| **1. 환경 검사** | Python 경로 및 필수 라이브러리(openpyxl) 체크 | 빌드 로그 | 테스트가 구동될 수 있는 **최소한의 인프라 안정성** 확인 |
| **2. QA 결과 분석** | Python을 이용해 엑셀 시트(TC_*) 내 결함 항목 분석 | `[FAIL_COUNT]` | 수천 개의 TC 중 즉시 수정이 필요한 **핵심 결함 지표** |
| **3. 리포트 생성** | 분석 데이터를 시각화한 HTML 리포트 자동 작성 | `qa_report.html` | 품질 현황을 한눈에 파악하게 하는 **전문적인 의사결정 도구** |
| **4. 결과 전파** | Slack API를 통한 결과 전송 및 파일 업로드 | Slack 메시지 | 결함 발생 즉시 팀에 알림으로써 **품질 개선 속도**를 극대화 |

---

## 3. 🎯 각 포트폴리오 구성의 의미

* **Python 스크립트 (`make_report.py`):** 수동 데이터 집계의 **휴먼 에러를 방지**하고, 엑셀 데이터를 정량적인 수치로 변환하여 객관적인 품질 리포트를 산출하는 분석 역량을 보여줍니다.
* **Jenkinsfile (Pipeline):** 파편화된 스크립트들을 하나의 유기적인 흐름으로 통합하여, 빌드부터 알림까지 이어지는 **전체 QA 프로세스를 설계 및 관리**할 수 있음을 의미합니다.
* **Slack 연동 및 파일 업로드:** 분석된 데이터를 로그로 남기는 데 그치지 않고, 팀원들이 가장 많이 사용하는 협업 도구로 직접 배달하는 **협업 최적화 마인드**를 담고 있습니다.

## 🛠 4. 설치 및 환경 설정 가이드 (Setup Guide)

### **① Git & GitHub 설정**
1.  [Git for Windows](https://git-scm.com/download/win)를 설치합니다.
2.  GitHub 리포지토리를 생성하고 본 프로젝트의 소스 코드와 `Tests/QA_Test.xlsx` 파일을 업로드합니다.

### **② Jenkins 환경 설정**
1.  Jenkins 설치 후 관리 > Plugins에서 **Slack Notification** 및 **HTML Publisher** 플러그인을 설치합니다.
2.  Jenkins 관리 > Credentials에 Slack 봇 토큰(Token)을 등록합니다.
3.  새 Pipeline 아이템을 생성하고, 'Pipeline script from SCM' 메뉴에서 GitHub 주소를 연동합니다.

### **③ Python 및 라이브러리**
1.  [Python 3.12](https://www.python.org/) 설치 시 `Add Python to PATH`를 반드시 체크합니다.
2.  터미널(CMD)에서 `pip install openpyxl` 명령어로 라이브러리를 설치합니다.
